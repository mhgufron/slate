    Introduction

    Welcome to the Kittn API! You can use our API to access Kittn API endpoints, which can get information on various cats, kittens, and breeds in our database.

    We have language bindings in Shell, Ruby, Python, and JavaScript! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API's documentation.

    Authentication

    Kittn uses API keys to allow access to the API. You can register a new Kittn API key at our developer portal.

    Kittn expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: meowmeowmeow

'8afcaefd25eb288d9e3754ffc21197990bbc98d18c352xxxx'}, verify=False).text

    Scenario 2:
    when sender type "Hi Woowa I want to ask about the product"

    Response:
    Hello
    thank you 

    post("https://116.203.92.59/api/add_bot", json = {'no_wa' : '6286484494xxx', 'data_input' :
'hello;match;text;Hello<ENTER>thank you;none', 'key' :
'8afcaefd25eb288d9e3754ffc21197990bbc98d18c352xxxx'}, verify=False).text

    Response:

    (text)

    This endpoint is used to upload bot.

    HTTP Request

    POST https://116.203.92.59/api/add_bot

    POST Parameters

    Parameter Description
    no_wa Client's whatsapp number.
    key Key partner.
    data_input The message id that you are looking for.

    Description for data_input Parameter

    Every variable in data_input is separated by ";". There are 5 variables:

    • Keyword: The specified keywords will read all lowercase letters. If there is more from 1 keyword with the same response, separate them with a "*" sign.

    • Match types: There are 3 types, namely match, partial and partial all.

      • match: Keywords are the same as input
      • partial: Keyword is similar to the initial input
      • partial all: Keyword mirip dengan apa yang ada pada input

    • Keyword types: There are 4 types namely text, text_image, image and script.

      • text: Response use text
      • image: Response use image
      • text_image: Response use text and image
      • script: The response will be processed in the script. The variable to be POSTed to The script is:
      • contact_name: Name / number of the sender, will send the name if already in contact and send a number if not yet in contact.
      • message: Message typed by the sender.
      • port: Client port
      • ip_server: IP server

    • Message: response message sent to the sender. Each newline then change the character newline with "". If you choose the image then upload the picture first then type the name of the image without extension.

    • Script: type url on your system's API.

    Response

    JSON Attribute Description
    (text) Returned text from bot.
